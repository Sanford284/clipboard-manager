# v0.2.0 Design Spec: Menu Bar & Window Behavior

## Overview

为 Clipboard Manager 添加 macOS 菜单栏托盘图标、窗口失焦隐藏、快捷键自定义功能。采用混合方案：菜单栏使用 Tauri 原生 tray 插件，快捷键修改对话框使用前端 WebView 实现。

## Feature 1: 菜单栏托盘图标

### 图标
- 22x22 像素单色模板图标（macOS template image），剪贴板形状
- 两种状态：监听中（正常显示）/ 已暂停（图标变灰或使用不同图标区分）

### 菜单结构
```
┌─────────────────────────────┐
│ ✓ 剪贴板监听                  │  勾选项，切换暂停/恢复监听
│ ─────────────────────────── │
│   打开剪贴板  (⌘⇧V)          │  打开主窗口，括号内显示当前快捷键
│   修改快捷键...               │  弹出快捷键设置对话框
│ ─────────────────────────── │
│   退出                       │  完全退出应用进程
└─────────────────────────────┘
```

### 行为
- **剪贴板监听**：勾选 = 正在监听。点击取消勾选暂停监听，再点恢复。暂停时停止记录新的剪贴板内容，应用仍在运行，历史记录保留。
- **打开剪贴板**：与全局快捷键行为一致，打开/聚焦主窗口。
- **退出**：完全关闭应用进程（`std::process::exit` 或 `app.exit()`）。

## Feature 2: 窗口失焦自动隐藏

- 监听 Tauri 窗口的 `WindowEvent::Focused(false)` 事件
- 当主窗口失焦时，自动调用 `window.hide()`
- 不影响现有粘贴流程（粘贴操作已在隐藏窗口后执行）
- 仅对主窗口生效，快捷键设置窗口不受影响

## Feature 3: 快捷键自定义

### 对话框窗口
- 独立 WebView 窗口，尺寸约 400x200，居中显示
- 不可调整大小，打开时主窗口不可交互（模态感）
- 窗口标题："修改快捷键"

### UI 交互
- 显示当前快捷键（如 `⌘⇧V`）
- 录入区域：提示"按下新的快捷键组合"
- 用户按下组合键后实时显示识别到的按键
- 三个操作按钮：
  - **保存**：保存新快捷键并立即生效
  - **取消**：关闭对话框不保存
  - **恢复默认**：恢复为 `⌘⇧V`（macOS）/ `Ctrl+Shift+V`（Windows）

### 持久化
- 快捷键保存到 SQLite `settings` 表（key: `shortcut`, value: 快捷键字符串如 `"CmdOrCtrl+Shift+V"`）
- 应用启动时从数据库读取 `shortcut` 设置，若无记录则使用默认值
- 保存后立即注销旧快捷键、注册新快捷键

## Technical Design

### Rust 后端变更

#### main.rs
- 引入 `tauri-plugin-tray`，初始化系统托盘
- 构建托盘菜单（剪贴板监听勾选项、打开剪贴板、修改快捷键、退出）
- 处理托盘菜单事件
- 在 `on_window_event` 中添加 `WindowEvent::Focused(false)` 处理，仅对主窗口调用 `window.hide()`
- 应用启动时从数据库读取快捷键设置，注册对应快捷键

#### commands.rs
- `get_shortcut() -> String`：从数据库读取当前快捷键
- `set_shortcut(shortcut: String)`：保存快捷键到数据库，注销旧快捷键，注册新快捷键
- `toggle_monitoring() -> bool`：切换监听状态，返回新状态

#### clipboard/mod.rs & clipboard/macos.rs
- 为 `ClipboardMonitor` 添加 `pause()`/`resume()` 方法
- 使用 `Arc<AtomicBool>` 标志控制监听循环是否记录新内容
- 暂停时轮询仍继续（保持线程活跃），但跳过内容比较和入库

#### storage/mod.rs
- `get_setting(key: &str) -> Option<String>`：从 `settings` 表读取配置
- `set_setting(key: &str, value: &str)`：写入/更新 `settings` 表配置

### 前端变更

#### 新增文件
- `src/ShortcutSettings.tsx`：快捷键设置组件
  - 监听 `keydown` 事件捕获组合键
  - 显示当前快捷键和新录入的快捷键
  - 调用 Tauri IPC `set_shortcut` / `get_shortcut`
  - 保存/取消/恢复默认按钮

#### 路由/窗口
- 快捷键设置使用独立窗口，通过 Tauri `WebviewWindow::new()` 创建
- 前端通过 URL 参数或路由区分主窗口和设置窗口

### 资源文件
- `src-tauri/icons/tray-icon.png`：托盘图标（正常状态）
- `src-tauri/icons/tray-icon-paused.png`：托盘图标（暂停状态）

### 依赖变更
- Cargo.toml 添加 `tauri-plugin-tray` 依赖（Tauri 2.x 中为 `tray-icon` 相关）
- tauri.conf.json 添加 tray 相关权限配置

### 数据库
- `settings` 表已存在，无需 migration
- 新增 key: `shortcut`，默认值 `"CmdOrCtrl+Shift+V"`
- 新增 key: `monitoring_enabled`，默认值 `"true"`（可选，用于启动时恢复监听状态）
